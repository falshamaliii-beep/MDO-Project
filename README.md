# MDO-Project


## Summary

This project explores how Microsoft Defender for Office 365 detects, analyzes, and mitigates email‑based threats such as phishing, malware, and malicious URLs. It demonstrates how AI‑driven insights and automated investigation workflows can strengthen organizational security and reduce response time.

## Background

Email remains the most common entry point for cyberattacks, including phishing, credential theft, and malware delivery. Organizations struggle to manually analyze large volumes of suspicious emails, and security teams often lack visibility into evolving threat patterns.

This project addresses:

The difficulty of detecting sophisticated phishing attempts

The need for automated analysis of malicious URLs and attachments

The challenge of understanding threat patterns across an organization

The importance of reducing investigation and response time

My motivation comes from studying Microsoft security products and wanting to understand how AI‑powered tools like MDO protect users in real‑world environments.

## How is it used?
The solution is designed for security analysts, IT administrators, and SOC teams who need visibility into email‑based threats.

Typical usage:

Collect threat data from Microsoft Defender for Office 365

Analyze phishing attempts, malware detections, and Safe Links/Safe Attachments events

Visualize threat patterns to understand attack frequency and severity

Support automated investigation and response workflows

This type of solution is useful in:

Enterprise environments

Organizations with high email traffic

Security operations centers (SOCs)

Any environment where phishing is a major risk

## Challenges

Ethical considerations:
